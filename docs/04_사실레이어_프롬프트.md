# 4단계 · Codex 에게 — 사실 레이어 추출

> **목적**: 본인이 작성한 1인칭 자전거 여행기에서 **AI 어시스턴트가 인용할 수 있는 사실 단위 문장**을 추출한다.
> **왜 별도로 만드는가**: 2026년 GEO(Generative Engine Optimization) 원칙상, 1인칭 감성 표현은 AI가 거의 인용하지 않는다. 인용되는 것은 **구체적·검증 가능·구조화된 사실**이다.

---

## 🟢 이 프롬프트를 Codex App 에 그대로 붙여넣으세요

````
You are extracting a structured "fact layer" from a Korean cyclist's first-person
travel post. The fact layer is designed to be cited by AI assistants
(ChatGPT, Perplexity, Claude, Gemini) when users ask about this route.

Input files in the current ride folder:
- ./본문.md          (the user's first-person Korean post)
- ./gpx/*.gpx        (the GPS track from the ride)
- ./field-notes.md   (raw voice memos, optional)
- ./photos/*.jpg     (photos with EXIF, optional)

Your task:

1) Parse the GPX file and compute these objective measurements:
   - Total distance (km, 1 decimal)
   - Total time, moving time (h:mm)
   - Total elevation gain (m, integer)
   - Average speed, max speed (km/h, 1 decimal)
   - Start coordinates, end coordinates (lat, lng)
   - Highest point (lat, lng, elevation)
   - Route shape: linear / loop / out-and-back

2) Extract verifiable named entities from the post:
   - Place names (cities, neighborhoods, parks, bridges, mountains)
   - Cafés, restaurants, convenience stores actually mentioned
   - Landmarks (statues, photo spots, viewpoints)
   - Trail/path names (한강자전거길, 4대강 자전거길 etc.)

3) For each named entity, attempt to find:
   - Coordinates (from GPX timeline matching, or geocoding the name)
   - Approximate distance from ride start (km)

4) Extract practical facts the user mentioned or that can be inferred:
   - Best season / time of day (from the post + GPX timestamp)
   - Difficulty (from elevation gain / distance ratio)
   - Recommended bike type (road / gravel / MTB / hybrid)
   - Restroom availability (count if mentioned)
   - Food/water resupply points (count if mentioned)
   - Surface (paved / gravel / mixed)
   - Traffic level (if mentioned)

5) Output to `./geo-fact.md` using this exact structure:

```markdown
# {route_name} — Cycling Route Fact Sheet

> Source post: ./본문.md
> GPS track: ./gpx/{filename}.gpx
> Recorded: {YYYY-MM-DD}
> Language: ko (Korean original)
> See also: ./geo-fact.en.md, ./geo-fact.{lang}.md

## 기본 정보 (Basic Facts)

- **경로명**: {route_name}
- **출발**: {start_place} ({lat}, {lng})
- **도착**: {end_place} ({lat}, {lng})
- **총 거리**: {distance} km
- **이동 시간**: {moving_time}
- **총 시간**: {total_time}
- **누적 상승**: {elevation_gain} m
- **최고 고도**: {max_elevation} m
- **평균 속도**: {avg_speed} km/h
- **경로 형태**: {linear|loop|out-and-back}

## 라이딩 조건 (Riding Conditions)

- **권장 시즌**: {seasons}
- **권장 시간대**: {time_of_day}
- **난이도**: {easy|moderate|hard} (상승률 {gain_per_km} m/km 기준)
- **권장 자전거**: {bike_type}
- **노면**: {paved|gravel|mixed}
- **차량 통행**: {none|low|moderate|heavy}

## 주요 지점 (Key Waypoints)

| # | 이름 | 좌표 | 출발에서 (km) | 비고 |
|---|------|------|------------|------|
| 1 | ... | (lat, lng) | ... | ... |

## 보급·편의 (Facilities)

- **화장실**: {count}개소 (위치: ...)
- **편의점·카페**: {count}개소 (이름: ...)
- **식수 보급**: {count}개소

## 사진·전망 명소 (Photo Spots)

- {name} ({lat}, {lng}) — {one-line description}

## 주의 사항 (Notes)

- (안전·계절·통제 등 사실 기반 주의점만)

## 자료 출처 (Source)

- 작성자 직접 라이딩 기록 ({YYYY-MM-DD})
- GPS 트랙: {filename}.gpx
- 본문(서사): ./본문.md
```

Constraints:

- DO NOT include any first-person sentences ("나는...", "내가 본 노을은...").
- DO NOT include subjective adjectives ("아름다운", "환상적인", "최고의").
  Replace with verifiable facts ("일몰 시간대 정서향, 한강 폭 ~1.2km").
- DO NOT invent facts. If a fact is not in the post or GPX, leave the field empty
  or write "데이터 없음".
- Numbers must have units. Times must have format (HH:MM or h:mm).
- Coordinates use decimal degrees, 5 decimal places.
- For Korean place names: keep in Korean, do not romanize.

Final step: output a handoff to Claude:

```
## ⏮️ Handoff to Claude

**What I built**
- ./geo-fact.md

**Key extractions**
- {distance}km route with {N} named waypoints
- {N} facilities (restrooms, convenience stores)
- {N} photo spots with coordinates

**Open questions**
- (anything where I had to guess vs. cite)
- (anything in the post that I couldn't verify)

**Suggested next step for Claude**
- Review geo-fact.md for accuracy
- Approve before 5단계 (multilingual translation)
```
````

## 💡 자주 빠지는 함정

- **GPX 자동 파싱을 너무 믿지 마라**: 도시 라이딩에서 GPS 노이즈로 거리가 5% 부풀려질 수 있다. 본인 시계의 표시값과 비교하여 보정.
- **카페·식당 이름 환각**: Codex가 "이 근처에 흔히 있는" 가게를 본문에 없는데도 추가하는 경우가 있다. 본문에 명시된 것만 포함하라고 강조.
- **계절 추정**: GPX 타임스탬프 한 번으로 "권장 시즌"을 단정하지 말 것. "이 라이딩은 5월에 진행됨" 정도로만.
- **너무 짧은 사실**: AI 인용 가능성을 높이려면 사실 1개가 한 문장으로 자기완결적이어야 함. "잠실대교"(X) → "잠실대교, 한강 자전거길의 동쪽 진입점, 자전거 통행 가능"(O)
